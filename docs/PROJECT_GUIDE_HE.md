# מדריך פרויקט: Network Snake

## מטרת הפרויקט

הפרויקט מממש משחק Snake מרובה משתתפים ברשת. יש שרת מרכזי שמנהל את כל חוקי המשחק, ולקוחות גרפיים שמתחברים אליו דרך TCP sockets. כל משתמש יכול להירשם, להתחבר, ליצור חדר משחק או להצטרף לחדר קיים.

## התאמה לדרישות החובה

1. תכנות מונחה עצמים:
   - `SnakeServer`
   - `ClientHandler`
   - `GameHub`
   - `GameRoom`
   - `Player`
   - `Bullet`
   - `UserStore`
   - `NetworkClient`
   - `SnakeClientApp`

2. תקשורת:
   - שימוש ב-`socket.AF_INET` וב-`socket.SOCK_STREAM`, כלומר TCP over IPv4.
   - השרת מרובה לקוחות: לכל לקוח נפתח `Thread` מסוג `ClientHandler`.
   - פרוטוקול התקשורת מבוסס JSON עם כותרת אורך של 4 בתים לפני כל הודעה.

3. מערכת הפעלה:
   - שימוש ב-Threads עבור לקוחות ולולאת המשחק.
   - שימוש במערכת הקבצים לשמירת משתמשים בקובץ `users.json`.

4. אבטחה:
   - החלפת מפתחות Diffie-Hellman בתחילת החיבור.
   - כל הודעות האפליקציה לאחר מכן מוצפנות.
   - הסיסמאות לא נשמרות כטקסט גלוי, אלא כ-hash עם salt באמצעות `PBKDF2-HMAC-SHA256`.
   - קיימות בדיקות שגיאות כדי שנפילת לקוח לא תפיל את השרת.

5. ממשק משתמש:
   - ממשק Tkinter עם מסך התחברות, לובי חדרים ומסך משחק.
   - המשחק מצויר ב-Canvas עם צבעים שונים לשחקנים, פירות ויריות.

## פרוטוקול תקשורת

כל הודעה נשלחת כך:

1. 4 בתים שמייצגים את אורך ההודעה.
2. גוף הודעה בפורמט JSON.
3. אחרי שלב החלפת המפתחות, גוף ההודעה מכיל מעטפת מוצפנת.

דוגמאות לפקודות:

- `register`: הרשמת משתמש.
- `login`: כניסת משתמש.
- `list_rooms`: בקשת רשימת חדרים.
- `create_room`: יצירת חדר.
- `join_room`: הצטרפות לחדר.
- `direction`: שינוי כיוון הנחש.
- `shoot`: ירייה.
- `game_state`: מצב המשחק שנשלח מהשרת ללקוחות.

## חוקי המשחק

- בכל חדר יכולים לשחק כמה שחקנים.
- כל לקוח יכול להיות רק בחדר אחד בכל רגע.
- הנחש מקבל נקודה על אכילת פרי ומתארך.
- פגיעה בקיר גורמת להפסד.
- פגיעת ראש בגוף של נחש גורמת להפסד.
- אחרי צבירת מספיק נקודות, השחקן יכול לירות.
- ירייה שפוגעת בנחש אחר הורגת אותו.
- המנצח הוא השחקן האחרון שנשאר בחיים.

## איך להריץ

בטרמינל ראשון:

```powershell
python -m snake_network.server.main
```

בטרמינל שני ושלישי:

```powershell
python -m snake_network.client.main
```

## הרצה ברשת ביתית

במחשב שמריץ את השרת, יש למצוא את כתובת ה-IPv4 המקומית:

```powershell
ipconfig
```

מחפשים כתובת כמו `192.168.1.25` או `10.0.0.8`.

במחשב השרת מריצים:

```powershell
python -m snake_network.server.main
```

באותו מחשב אפשר להריץ לקוח כך:

```powershell
python -m snake_network.client.main --host 127.0.0.1
```

במחשב אחר באותה רשת ביתית מריצים:

```powershell
python -m snake_network.client.main --host 192.168.1.25
```

יש להחליף את `192.168.1.25` בכתובת האמיתית של מחשב השרת. אם Windows Firewall מציג בקשה, צריך לאפשר ל-Python גישה ברשתות פרטיות.

## הצעות לבדיקות

- הרשמה: יצירת משתמש חדש. תוצאה צפויה: המשתמש נשמר ויכול להיכנס.
- כניסה: התחברות עם סיסמה נכונה. תוצאה צפויה: מעבר למסך הלובי.
- סיסמה שגויה: התחברות עם סיסמה לא נכונה. תוצאה צפויה: מתקבלת הודעת שגיאה.
- כמה לקוחות: הפעלת שני לקוחות. תוצאה צפויה: שניהם יכולים להצטרף לאותו חדר.
- כמה חדרים: יצירת שני חדרים. תוצאה צפויה: כל חדר מנהל משחק עצמאי.
- אכילת פרי: הנחש עובר על פרי. תוצאה צפויה: הניקוד עולה והנחש מתארך.
- פגיעה בקיר: הנחש יוצא מגבולות הלוח. תוצאה צפויה: השחקן נפסל.
- ירי: שחקן עם מספיק נקודות לוחץ Space. תוצאה צפויה: נוצרת ירייה שמתקדמת בלוח.

## הערת אבטחה חשובה

ההצפנה בפרויקט מיועדת להדגמה לימודית של עקרונות הצפנה, החלפת מפתחות ואימות שלמות הודעה. במערכת אמיתית מומלץ להשתמש ב-TLS או בספריית הצפנה מקצועית.
